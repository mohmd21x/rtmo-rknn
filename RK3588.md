# RK3588 Deployment Guide

Run RTMO pose estimation on Rockchip RK3588 (NPU) with the no-NMS RKNN pipeline.

## What runs where

| Step | PC (x86) | RK3588 board |
|------|----------|--------------|
| Export no-NMS ONNX + decoder YAML | `export_no_nms.py` | optional (slow) |
| ONNX → RKNN conversion | `convert_rknn.py` / `convert_all.sh` | optional (needs `rknn-toolkit2` aarch64 wheel) |
| NPU inference + DCC decode + NMS | simulator only (rebuilds from ONNX; scores differ) | **`rknnlite2` + `rtmo_rknn.py`** |

**Recommended:** convert on PC, copy `.rknn` + decoder YAML to the board, infer with `rknnlite2`.

---

## Project checklist (before moving to the board)

On your dev PC, from the repo root:

```bash
cd rtmo-rknn

# 1) Export NMS-free ONNX + decoder params (once per model)
python convert/export_no_nms.py --model rtmo/rtmo-s.onnx
python convert/export_no_nms.py --model rtmo/rtmo-m.onnx

# 2) Convert to RKNN (fp16 + int8)
bash convert/convert_all.sh
# or single model:
python convert/convert_rknn.py \
  --model convert/no_nms_onnx/rtmo-m-no-nms.onnx \
  --output convert/models/rtmo-m.fp16.rknn \
  --quant fp16 --target rk3588

# 3) Verify board package (all .rknn + decoder YAML present)
python test/check_board_ready.py
```

Expected artifacts:

```text
convert/no_nms_onnx/rtmo-s-no-nms.onnx
convert/no_nms_onnx/rtmo-m-no-nms.onnx
convert/decoder/rtmo_s_dcc_decoder_params.yml   # ~11 MB
convert/decoder/rtmo_m_dcc_decoder_params.yml   # ~16 MB
convert/models/rtmo-s.fp16.rknn                 # ~22 MB
convert/models/rtmo-s.int8.rknn                 # ~12 MB
convert/models/rtmo-m.fp16.rknn                 # ~48 MB
convert/models/rtmo-m.int8.rknn                 # ~26 MB
```

## Copy to RK3588

### Minimal package (inference only)

Copy these to the board (e.g. `/home/rock/rtmo-rknn/`), preserving directories:

```bash
BOARD=rock@<board-ip>
REMOTE=/home/rock/rtmo-rknn

rsync -av --progress \
  --include='*/' \
  --include='rtmo_rknn.py' \
  --include='requirements-rk3588.txt' \
  --include='RK3588.md' \
  --include='convert/decoder/***' \
  --include='convert/models/*.rknn' \
  --include='benchmark/benchmark.py' \
  --include='test/compare_video.py' \
  --include='sample-data/***' \
  --include='rtmo/video/***' \
  --exclude='*' \
  ./ "${BOARD}:${REMOTE}/"
```

Decoder YAML is required at runtime (auto-resolved from model name `rtmo-s` / `rtmo-m`).

## Board setup

SSH into the RK3588:

```bash
ssh rock@<board-ip>
cd ~/rtmo-rknn

python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements-rk3588.txt

# Install Rockchip rknnlite2 wheel (version must match your OS + Python)
pip install /path/to/rknnlite2-*.whl
```

Verify NPU runtime:

```bash
python -c "from rknnlite.api import RKNNLite; print('rknnlite2 OK')"
```

## Test on RK3588 (NPU)

> **Important:** If inference segfaults on the board, re-export ONNX and **re-convert** `.rknn` on a PC with rknn-toolkit2.  
> Old models included an ONNX **TopK** op that crashes RK3588 NPU at `rknn_run`.  
> Run on PC: `python convert/export_no_nms.py --model rtmo/rtmo-m.onnx` then `convert/convert_all.sh`, then rsync new `convert/models/*.rknn` to the board.

Always use **`--use_simulator false`** on the board so `rknnlite2` loads the pre-built `.rknn` via `load_rknn`.

### Smoke test (one frame)

```bash
python -c "
import cv2
from rtmo_rknn import RTMO_RKNN

cap = cv2.VideoCapture('rtmo/video/Oldest video ever - 1888 [tc-L9_4jGc4].mp4')
ok, frame = cap.read()
assert ok, 'video missing'

m = RTMO_RKNN(
    model_path='convert/models/rtmo-m.fp16.rknn',
    use_simulator=False,
    target='rk3588',
    device_id=0,
)
_, boxes, scores, keypoints, kpt_scores = m(frame)
print(f'detections: {len(boxes)}')
if len(scores):
    print(f'top score: {scores.max():.3f}')
    print(f'keypoints shape: {keypoints.shape}')
m.release()
"
```

### Video compare on board (RKNN only — recommended)

Full ONNX (`rtmo-m.onnx`) uses mmdeploy NMS custom ops and often **segfaults** on RK3588.
`compare_video.py` auto-enables RKNN-only mode when `--rknn_backend device` (no `--onnx` needed).

```bash
python3 test/compare_video.py \
  --video rtmo/video/danial-fall-fast-2-1.mp4 \
  --rknn convert/models/rtmo-m.fp16.rknn \
  --rknn_backend device \
  --use_simulator false \
  --score_threshold 0.3 \
  --output test/compare_board.mp4
```

Optional smoke test (first few frames, no video writer):

```bash
python3 test/smoke_rknn_device.py \
  --rknn convert/models/rtmo-m.fp16.rknn \
  --video rtmo/video/danial-fall-fast-2-1.mp4 \
  --frames 5
```

Use `--display` only if the board has a desktop/X11 session; omit it over SSH.

For ONNX vs RKNN visual compare, run on a PC with `--rknn_backend onnx` instead.

RKNN device input is **uint8 4D NHWC**; rknnlite applies mean/std from conversion (`mean=0`, `std=1` → float 0–255, matching ONNX). Do not set `RTMO_RKNN_INPUT=float32` (causes segfault on rknnlite 2.3.x).

> Side-by-side full ONNX on the board (`--with_onnx`) is not recommended.

### Benchmark speed + accuracy vs ONNX (if ONNX installed on board)

```bash
source .venv/bin/activate   # required — system pip is blocked on Ubuntu/Debian

# rtmo-m fp16
python benchmark/benchmark.py \
  --onnx rtmo/rtmo-m.onnx \
  --rknn convert/models/rtmo-m.fp16.rknn \
  --use_simulator false \
  --target rk3588 \
  --frames 200 \
  --score_threshold 0.3 \
  --output benchmark/results_rtmo-m_fp16_board.csv

# rtmo-m int8
python benchmark/benchmark.py \
  --onnx rtmo/rtmo-m.onnx \
  --rknn convert/models/rtmo-m.int8.rknn \
  --use_simulator false \
  --target rk3588 \
  --frames 200 \
  --score_threshold 0.3 \
  --output benchmark/results_rtmo-m_int8_board.csv

# rtmo-s fp16
python benchmark/benchmark.py \
  --onnx rtmo/rtmo-s.onnx \
  --rknn convert/models/rtmo-s.fp16.rknn \
  --use_simulator false \
  --target rk3588 \
  --frames 200 \
  --output benchmark/results_rtmo-s_fp16_board.csv
```

On NPU, default **`--score_threshold 0.3`** matches ONNX (`rtmo_gpu` filters `score > 0.3` after embedded NMS). For INT8 models with extra false positives, try `0.4`–`0.5`.

## Troubleshooting

| Symptom | Fix |
|---------|-----|
| `rknnlite2 is required for on-device mode` | Install Rockchip `rknnlite2` wheel on the board |
| `Failed to init RKNN device runtime` | Check `sudo cat /sys/kernel/debug/rknpu/version`, reboot, ensure `.rknn` was built with `--target rk3588` |
| `Failed to load RKNN model` | Re-copy `.rknn`; verify file size matches PC build |
| `DCC decoder params not found` | Copy `convert/decoder/rtmo_*_dcc_decoder_params.yml` |
| Segfault at first RKNN inference (no Python traceback) | Old `.rknn` contains **TopK** — re-run `export_no_nms.py` + `convert_rknn.py` (new export cuts before TopK; CPU does sort) |
| `Expected 5/6 RKNN outputs` or misaligned boxes/keypoints at top of frame | Re-run `export_no_nms.py` then `convert_rknn.py` |
| Bboxes OK but keypoints random / far from ONNX | Re-run `export_no_nms.py` to refresh decoder YAML (`x_fc`/`y_fc` weights), then redeploy `convert/decoder/rtmo_*_dcc_decoder_params.yml`. DCC decode uses `flatten_priors_640` from YAML (raw anchor via `sort_indices`), not the network `priors` output |
| RKNN shows 2+ dets but ONNX shows 1 | RKNN now uses two-stage filtering like ONNX: NMS at score≥0.15, then keep score≥0.3 (default). Do not use `--score_threshold 0.05`. For INT8 false positives, try `--score_threshold 0.4` or `0.5` |
| PC simulator: 0 detections | Lower `--score_threshold` (e.g. `0.2`) or test on board with `--use_simulator false` |
| PC: `load_rknn not support inference on the simulator` | Expected — simulator rebuilds from `convert/no_nms_onnx/*.onnx` automatically |
| `externally-managed-environment` on `pip install` | Use the project venv (`python3 -m venv .venv && source .venv/bin/activate`); do not `pip install` into system Python |
| `Unsupported run platform: Linux aarch64` | Fixed in `rtmo_rknn.py`: on-board `rknnlite2` must call `init_runtime()` without `target` |

## Git: what is tracked

See `.gitignore` (whitelist). Tracked essentials:

- Source: `rtmo_rknn.py`, `convert/*.py`, `benchmark/`, `test/`
- Weights: `rtmo/rtmo-s.onnx`, `rtmo/rtmo-m.onnx`, `convert/decoder/*.yml`
- **RKNN models:** `convert/models/*.rknn` (fp16 + int8 for rtmo-s and rtmo-m)
- Calibration: `sample-data/`

**Not tracked** (regenerate locally): `convert/no_nms_onnx/`, `convert/models/dataset.txt`, `check*.onnx`, benchmark CSVs, test output videos, `rtmo_s-SAMPLE/`.

Before copying to the board, run:

```bash
python test/check_board_ready.py
```
